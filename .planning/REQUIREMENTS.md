# Требования к проекту BeautyLanding

> Детальные функциональные и нефункциональные требования для MVP

## 🎯 Цели и ограничения

### Бизнес-цели
- Запустить MVP за 2-4 недели
- Получить первых 5 платных пользователей
- Валидировать спрос на продукт
- Создать базу для дальнейшего развития

### Технические ограничения
- Бюджет: $0 (только free tier сервисов)
- Разработчик: начинающий уровень
- Время: 42-140 часов разработки
- Стек: React + Node.js + PostgreSQL

### Пользовательские ограничения
- Целевая аудитория: косметологи без технических навыков
- Время на создание лендинга: < 30 минут
- Интерфейс должен быть интуитивно понятным
- Мобильная адаптация обязательна

## 👥 Пользовательские роли

### 1. Гость (неавторизованный пользователь)
**Может:**
- Просматривать главную страницу BeautyLanding
- Смотреть примеры шаблонов
- Просматривать цены
- Регистрироваться

**Не может:**
- Создавать лендинги
- Редактировать шаблоны
- Публиковать сайты

### 2. Пользователь (зарегистрированный)
**Может:**
- Все что гость +
- Создавать лендинги из шаблонов
- Редактировать контент (текст, фото, цены)
- Предпросматривать результат
- Экспортировать HTML (после оплаты)
- Публиковать на поддомене (после оплаты)

**Не может:**
- Создавать собственные шаблоны
- Изменять структуру шаблона
- Получать доступ к чужим лендингам

### 3. Администратор (будущая роль)
**Может:**
- Все что пользователь +
- Управлять шаблонами
- Просматривать статистику
- Модерировать контент
- Управлять пользователями

## 📋 Функциональные требования

### FR-1: Аутентификация и авторизация

#### FR-1.1: Регистрация
**Приоритет:** Must Have  
**Описание:** Пользователь может зарегистрироваться в системе

**Acceptance Criteria:**
- [ ] Форма регистрации с полями: email, пароль, имя
- [ ] Валидация email (формат)
- [ ] Валидация пароля (минимум 8 символов)
- [ ] Проверка уникальности email
- [ ] Отправка письма подтверждения (опционально для MVP)
- [ ] Автоматический вход после регистрации
- [ ] Сообщения об ошибках на русском языке

**Технические детали:**
- Хеширование паролей через bcrypt
- JWT токены для сессий
- Хранение в PostgreSQL

#### FR-1.2: Вход в систему
**Приоритет:** Must Have  
**Описание:** Зарегистрированный пользователь может войти

**Acceptance Criteria:**
- [ ] Форма входа: email + пароль
- [ ] Валидация данных
- [ ] Генерация JWT токена
- [ ] Сохранение токена в localStorage
- [ ] Редирект на дашборд после входа
- [ ] Обработка неверных данных

#### FR-1.3: Восстановление пароля
**Приоритет:** Should Have  
**Описание:** Пользователь может восстановить забытый пароль

**Acceptance Criteria:**
- [ ] Форма "Забыли пароль?"
- [ ] Отправка ссылки на email
- [ ] Страница сброса пароля
- [ ] Установка нового пароля

### FR-2: Управление шаблонами

#### FR-2.1: Просмотр шаблонов
**Приоритет:** Must Have  
**Описание:** Пользователь видит доступные шаблоны

**Acceptance Criteria:**
- [ ] Галерея из минимум 3 шаблонов
- [ ] Превью каждого шаблона
- [ ] Название и описание шаблона
- [ ] Кнопка "Использовать шаблон"
- [ ] Адаптивная сетка (grid)

**Шаблоны для MVP:**
1. **"Классика"** — универсальный шаблон для косметолога
2. **"Инъекции"** — для инъекционной косметологии
3. **"Минимализм"** — лаконичный дизайн

#### FR-2.2: Выбор шаблона
**Приоритет:** Must Have  
**Описание:** Пользователь создает новый лендинг из шаблона

**Acceptance Criteria:**
- [ ] Клик по шаблону открывает редактор
- [ ] Создается копия шаблона для пользователя
- [ ] Лендинг сохраняется в базе данных
- [ ] Редирект в редактор

### FR-3: Редактор лендинга

#### FR-3.1: Редактирование текста
**Приоритет:** Must Have  
**Описание:** Пользователь может изменять текстовый контент

**Acceptance Criteria:**
- [ ] Клик по тексту активирует режим редактирования
- [ ] Inline редактирование (без модальных окон)
- [ ] Автосохранение изменений
- [ ] Поддержка многострочного текста
- [ ] Ограничение длины для заголовков

**Редактируемые элементы:**
- Заголовки (H1, H2, H3)
- Описания услуг
- Цены
- Контактная информация
- Призывы к действию (CTA)

#### FR-3.2: Загрузка изображений
**Приоритет:** Must Have  
**Описание:** Пользователь может загружать свои фото

**Acceptance Criteria:**
- [ ] Клик по изображению открывает загрузчик
- [ ] Поддержка форматов: JPG, PNG, WebP
- [ ] Максимальный размер: 5MB
- [ ] Автоматическое сжатие и оптимизация
- [ ] Превью перед загрузкой
- [ ] Загрузка в Cloudinary
- [ ] Обработка ошибок

**Типы изображений:**
- Фото мастера
- Фото кабинета/салона
- Результаты работ (до/после)
- Логотип (опционально)

#### FR-3.3: Редактирование услуг и цен
**Приоритет:** Must Have  
**Описание:** Управление списком услуг

**Acceptance Criteria:**
- [ ] Добавление новой услуги
- [ ] Редактирование названия услуги
- [ ] Редактирование цены
- [ ] Редактирование описания
- [ ] Удаление услуги
- [ ] Изменение порядка услуг (drag & drop)

**Структура услуги:**
```json
{
  "id": "uuid",
  "name": "Чистка лица",
  "description": "Глубокое очищение пор",
  "price": 3000,
  "currency": "₽",
  "duration": "60 мин"
}
```

#### FR-3.4: Настройка контактов
**Приоритет:** Must Have  
**Описание:** Редактирование контактной информации

**Acceptance Criteria:**
- [ ] Телефон (с маской ввода)
- [ ] WhatsApp (опционально)
- [ ] Telegram (опционально)
- [ ] Instagram (опционально)
- [ ] Адрес (опционально)
- [ ] Email (опционально)

#### FR-3.5: Настройка формы записи
**Приоритет:** Must Have  
**Описание:** Конфигурация формы для записи клиентов

**Acceptance Criteria:**
- [ ] Выбор способа связи: WhatsApp, Telegram, Email
- [ ] Настройка полей формы (имя, телефон, услуга)
- [ ] Текст кнопки отправки
- [ ] Сообщение после отправки

**Интеграции:**
- WhatsApp: открытие чата с предзаполненным сообщением
- Telegram: отправка в бот или открытие чата
- Email: отправка на указанный email

#### FR-3.6: Галерея работ (До/После)
**Приоритет:** Must Have  
**Описание:** Секция с результатами работ

**Acceptance Criteria:**
- [ ] Загрузка пар фото (до/после)
- [ ] Минимум 3 пары для MVP
- [ ] Слайдер или сетка для отображения
- [ ] Подписи к фото (опционально)
- [ ] Удаление фото

### FR-4: Предпросмотр и публикация

#### FR-4.1: Предпросмотр
**Приоритет:** Must Have  
**Описание:** Просмотр лендинга перед публикацией

**Acceptance Criteria:**
- [ ] Кнопка "Предпросмотр" в редакторе
- [ ] Открытие в новой вкладке или модальном окне

#### FR-4.2: Экспорт HTML
**Приоритет:** Must Have  
**Описание:** Скачивание готового лендинга

**Acceptance Criteria:**
- [ ] Кнопка "Скачать HTML"
- [ ] Генерация standalone HTML файла
- [ ] Встроенные стили (inline CSS)

**Технические требования:**
- Один HTML файл
- Размер < 2MB

**Приоритет:** Should Have  
**Описание:** Хостинг лендинга на beautylanding.app

- [ ] Проверка доступности поддомена
- [ ] Публикация одним кликом
- [ ] SSL сертификат (автоматически)
- [ ] Обновление при изменениях
- [ ] Доступно только после оплаты

## 🎨 Нефункциональные требования

### NFR-1: Производительность

**NFR-1.1: Скорость загрузки**
- Главная страница: < 2 секунды
- Редактор: < 3 секунды
- Опубликованный лендинг: < 2 секунды
- Метрика: Lighthouse Performance Score > 90

**NFR-1.2: Отзывчивость интерфейса**
- Время отклика на действия: < 100ms
- Автосохранение: каждые 3 секунды
- Загрузка изображений: с прогресс-баром

### NFR-2: Безопасность

**NFR-2.1: Аутентификация**
- Хеширование паролей (bcrypt, cost 10)
- JWT токены с истечением (7 дней)
- HTTPS обязательно
- Защита от CSRF

**NFR-2.2: Валидация данных**
- Валидация на клиенте и сервере
- Санитизация пользовательского ввода
- Защита от XSS
- Защита от SQL injection (через ORM)

**NFR-2.3: Приватность**
- Лендинги пользователя видны только ему
- Безопасное хранение токенов
- Соответствие GDPR (базовое)

### NFR-3: Юзабилити

**NFR-3.1: Простота использования**
- Создание лендинга без инструкции
- Интуитивный интерфейс редактора
- Подсказки и tooltips
- Отмена действий (Ctrl+Z)

**NFR-3.2: Доступность (a11y)**
- Семантический HTML
- ARIA атрибуты
- Навигация с клавиатуры
- Контрастность текста (WCAG AA)

**NFR-3.3: Мобильная адаптация**
- Responsive дизайн
- Touch-friendly элементы (минимум 44x44px)
- Тестирование на iOS и Android
- Поддержка жестов

### NFR-4: Совместимость

**NFR-4.1: Браузеры**
- Chrome 100+ ✅
- Safari 15+ ✅
- Firefox 100+ ✅
- Edge 100+ ✅
- Мобильные браузеры ✅

**NFR-4.2: Устройства**
- Desktop (1920x1080 и выше)
- Laptop (1366x768)
- Tablet (768x1024)
- Mobile (375x667 и выше)

### NFR-5: Масштабируемость

**NFR-5.1: Пользователи**
- MVP: до 100 пользователей
- Будущее: до 10,000 пользователей

**NFR-5.2: Данные**
- Лендингов на пользователя: до 10
- Изображений на лендинг: до 20
- Размер базы данных: до 1GB (free tier)

### NFR-6: Надежность

**NFR-6.1: Доступность**
- Uptime: > 99% (цель)
- Резервное копирование: ежедневно
- Восстановление: < 1 час

**NFR-6.2: Обработка ошибок**
- Graceful degradation
- Понятные сообщения об ошибках
- Логирование ошибок (Sentry)

### NFR-7: Поддерживаемость

**NFR-7.1: Код**
- Чистый и читаемый код
- Комментарии на русском языке
- Модульная архитектура
- Покрытие тестами > 70%

**NFR-7.2: Документация**
- README с инструкциями по запуску
- API документация
- Комментарии в коде
- Changelog
